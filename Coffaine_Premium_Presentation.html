01 · Overview

Problem & Vision

Faculty Coffee — 14 Piccadilly Arcade, Birmingham — needed a complete digital transformation

⚡
The Problem
No online ordering — customers had to visit in person only
Inventory tracked manually on paper — error-prone and slow
No analytics — zero visibility into sales trends or top products
No customer communication platform — inquiries completely untracked
Staff hiring done manually — no digital application system
Store open/closed communicated only physically — no automation
☕
Our Vision
A cinematic public site where customers browse, order, and interact
A powerful admin dashboard with 12 modules — full operational control
Dual AI: Sophie for customers, CaffAIne for business intelligence
Real-time inventory with recipe-based auto-deduction on every order
Voice search in English and Arabic — bilingual dictionary built-in
Auto open/close: Mon–Fri 07:30–17:00, Sat 09:00–18:00, Sun 10:00–16:00

Dev Tools
Concurrently + dotenv

11 · Orders Module

Real-Time Order Management

Live countdown, status lifecycle, atomic stock-checked transactions

📋 Order Lifecycle
Statuses: preparing → ready → completed
1
POST /api/orders — BEGIN transaction, check stock per ingredient
2
Insert order: status='preparing', estimated_ready_at = NOW() + 2 MIN
3
Admin sees TIMESTAMPDIFF(SECOND, NOW(), estimated_ready_at) countdown
4
Extend: PUT /api/extend-order/:id → adds N minutes to timer
5
Mark ready: PUT /api/mark-ready/:id → notification log fired
6
Items fetched: /api/order-items/:orderId with JOIN to menu_items
7
Total recalculated server-side if client total is lower than expected
💳 Data Captured per Order
Everything stored and linked
customer_name, phone (required), email (optional)
order_type: takeaway / dine-in / delivery
delivery_address if delivery selected
Each item: product_id, item_name, quantity, price per row
Add-ons: separate order_items rows with addon price
Auto-fix: 0-price addons looked up from addons table
Inventory: GREATEST(quantity - deduct, 0) prevents negatives
Recipe deduction: quantity_required × qty + addon inventory_id

📋 Audit Log Middleware
req.logAdminAction on every route
Reads x-admin-email + x-admin-name headers on every request

👥 Role-Based Access
super_admin vs admin

Regular admins: no audit logs, no critical feedback panel
🛡 Data Integrity
Server-side validation everywhere

Prepared statements across all MySQL queries — SQL injection proof
CORS: only localhost origins allowed via regex pattern
Order: customer_name + phone + cartItems required — 400 if missing
