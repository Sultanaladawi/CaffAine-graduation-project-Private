01 · Overview

Problem & Vision

Faculty Coffee — real café at 14 Piccadilly Arcade, Birmingham — needed a complete digital transformation

⚡
The Problem
No online ordering — customers had to visit in person only
Inventory tracked manually on paper — error-prone and slow
No analytics — zero visibility into sales trends or top products
No customer communication platform — inquiries untracked
Staff hiring done manually — no digital application system
Store open/closed status communicated only physically
☕
Our Vision
A cinematic public site where customers browse, order, and interact
A powerful admin dashboard with 12 modules — full operational control
Dual AI assistants: Sophie for customers, CaffAIne for business intelligence
Real-time inventory with recipe-based auto-deduction on every order
Voice search in English and Arabic — bilingual dictionary built-in
Auto open/close with schedule: Mon–Fri 07:30–17:00, Sat 09:00–18:00, Sun 10:00–16:00
02 · Architecture

Technology Stack

Modern, scalable, production-ready — chosen for performance and maintainability

Frontend
React.js 18
React Router v7, Context API (CartContext + StoreContext + AdminContext), custom Hooks (useReveal, useAdminAuth), Lenis smooth scroll, animated loading screen, custom cursor
Backend
Node.js + Express 5
40+ RESTful API endpoints, Multer file uploads (10MB, images only), MySQL2 connection pool (10 connections), CORS, audit log middleware on every request
Database
MySQL + Auto-Migration
14 tables with FK constraints, auto schema migration on startup, connection pool, prepared statements (SQL injection protected), atomic transactions for orders
Artificial Intelligence
OpenAI GPT-4o-mini
Dual-mode: Sophie (customer) + CaffAIne (admin). GitHub Models fallback (Azure inference). 19 parallel DB queries for admin context. 45s timeout with AbortController
UI / Icons
Lucide React + CSS Modules
Lucide icons in admin, Font Awesome in public site, CSS Modules for scoped styles, custom design system, 30+ tag types with color-coded gradients
Dev Tools
Concurrently + dotenv
Parallel frontend + backend dev, .env + .env.development config, jsPDF + autoTable for PDF export, axios for admin API calls, supabaseClient.js for storage

19 · UX Details & Challenges

Premium UX & Real Challenges Solved

The thoughtful details and real engineering problems we faced during development

✨ UX Details Built
Every detail from the actual codebase
Lenis Smooth Scroll: @studio-freight/lenis with duration 1.25s and custom easing function
Custom Cursor: #cursor-dot + #cursor-ring — branded, replaces browser default
Loading Screen: Custom LoadingScreen component with onComplete callback → fades into hero
Mouse Parallax: Hero image moves ±10px/8px based on cursor position in section
useReveal Hook: IntersectionObserver-based — adds 'vis' class when element enters viewport
Scroll Progress Bar: #scroll-progress element shows reading position
Voice Search: Web Speech API + bilingual Arabic dictionary in Menu + Sophie
MobileNav: Separate mobile navigation component for responsive layout
⚡ Real Challenges Solved
Actual problems encountered during development
0-Price Addons: Migration script fixes historical order_items where price=0 by looking up addons table
Category Migration: Legacy string IDs (espresso, cold, food) mapped to numeric IDs — smart recovery keywords
AI Context Size: 19 queries for CaffAIne — used Promise.allSettled() so one failure doesn't break others
Dual AI Provider: Auto-detects GitHub token (ghp_/github_) vs OpenAI key → switches baseURL accordingly
Arabic Numerals: convertNumerals() converts ١٢٣ → 123 for inventory inputs from Arabic users
FK Conflict on Delete: Product delete fails if in orders → returns 400 with "mark as Unavailable instead"
Addon Price Fix: On startup, UPDATE addons SET price=0.50 WHERE price=0 — prevents free add-ons
