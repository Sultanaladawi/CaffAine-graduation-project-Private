01 · Overview

Problem & Vision

Faculty Coffee — real café at 14 Piccadilly Arcade, Birmingham — needed a complete digital transformation

⚡
The Problem
No online ordering — customers had to visit in person only
Inventory tracked manually on paper — error-prone and slow
No analytics — zero visibility into sales trends or top products
No customer communication platform — inquiries untracked
Staff hiring done manually — no digital application system
Store open/closed status communicated only physically
☕
Our Vision
A cinematic public site where customers browse, order, and interact
A powerful admin dashboard with 12 modules — full operational control
Dual AI assistants: Sophie for customers, CaffAIne for business intelligence
Real-time inventory with recipe-based auto-deduction on every order
Voice search in English and Arabic — bilingual dictionary built-in
Auto open/close with schedule: Mon–Fri 07:30–17:00, Sat 09:00–18:00, Sun 10:00–16:00

11 · Orders Module

Real-Time Order Management

Live countdown timer, status lifecycle, stock-checked transactions, and order item details

📋 Order Lifecycle
Statuses: preparing → ready → completed
1
Customer places order — POST /api/orders with transaction
2
Stock checked per ingredient before insert — throws error if insufficient
3
Order inserted with status='preparing', estimated_ready_at = NOW() + 2 MIN
4
Admin sees live countdown: TIMESTAMPDIFF(SECOND, NOW(), estimated_ready_at)
5
Admin can extend: PUT /api/extend-order/:id — adds N minutes to timer
6
Mark ready: PUT /api/mark-ready/:id — triggers notification log (SMS + email)
7
Full order items fetched from /api/order-items/:orderId with JOIN to menu_items
💳 Checkout Data Captured
Everything stored per order
customer_name, phone (required), email (optional)
order_type: takeaway / dine-in / delivery
delivery_address (if delivery selected)
Each cart item: product_id, item_name, quantity, price
Add-ons: separate order_items rows with addon price
Auto-correct: 0-price addons looked up from addons table
total_amount recalculated server-side if client total is lower
Inventory auto-deducted: recipes × quantity + addon inventory_id

13 · Inventory

Inventory & Recipe System

Smart stock tracking — recipe-linked auto-deduction on every order, zero manual counting

📦 Stock Control
Real-time ingredient tracking
1
Inventory table: item_name, quantity, unit, min_threshold
2
Admin sets threshold — Dashboard and Inventory flag items below it
3
PUT /api/update-stock-item/:id — admin manually adjusts quantity + unit
4
convertNumerals() — supports Arabic numeral input (٠١٢٣٤٥٦٧٨٩)
5
CaffAIne AI reads critical + approaching low stock (within 150% of threshold)
6
Low-stock check in products query: marks isOutOfStock=true if any ingredient below required
🧪 Recipe System
Automated deduction on every sale
1
recipes table: menu_item_id, inventory_id, quantity_required
2
GET /api/products/:id/recipe — admin views ingredient map per product
3
POST /api/products/:id/recipe — clears old recipe, inserts new ingredients array
4
On order: for each item, deduct quantity_required × qty from inventory
5
On order: for each addon, deduct 1 × qty from addon's inventory_id
6
GREATEST(quantity - deduct, 0) — prevents negative stock values

17 · Security

Security & Access Control

Role-based authentication, server-side validation, and complete audit transparency

🔐 Authentication System
POST /api/admin/login — team credentials in server.js
4 team accounts: omar@coffee.com (super_admin), sultan@coffee.com, mohammad@coffee.com, bashar@coffee.com (all admin)
AdminContext stores admin object: id, email, name, role
AdminRoute component wraps all /admin/* routes — redirects to login if no admin in context
useAdminAuth hook + useAdminContext for role checking throughout
Every login action inserted into admin_logs automatically
📋 Audit Log System
Every request passes through logAdminAction middleware
Middleware reads x-admin-email + x-admin-name headers on every request
req.logAdminAction(action, details) available on every route
admin_logs table: admin_email, admin_name, action, details, created_at
GET /api/admin/logs — latest 200 entries returned to Leader Dashboard
Searchable by name, action, or details in real-time
👥 Role-Based Access
Two roles: super_admin (Omar) + admin (Sultan, Mohammad, Bashar)
LeaderDashboard: if admin.role !== 'super_admin' → navigate('/admin/dashboard')
Leader auto-refreshes every 15s — always current
Regular admins: no access to audit logs or critical feedback panel
All role checks happen client-side in AdminContext + server-side via headers
🛡 Data Integrity
Server-side validation on every endpoint
Multer: fileFilter allows only jpeg/jpg/png/webp/gif — rejects others
Prepared statements across all MySQL queries — SQL injection protected
CORS: only localhost origins allowed (dev) via regex pattern
Order validation: customer_name + phone + cartItems required — 400 if missing
