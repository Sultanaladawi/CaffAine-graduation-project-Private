# وثيقة توثيق المشروع: نظام CaffAIne للذكاء الإداري
**التاريخ:** 11 مايو 2026
**عنوان المشروع:** CaffAIne - منظومة إدارة ومخابرات الأعمال لمقاهي Specialty Coffee
**رئيس الفريق:** عمر العجارمة
**أعضاء الفريق الأساسي:** سلطان العدوي، محمد الحديدي، بشار الدباس

---

## 1. نظرة عامة على المشروع
CaffAIne هو نظام متكامل لإدارة العمليات (Full-Stack) مصمم خصيصاً لمقهى **Faculty Coffee** في برمنغهام، المملكة المتحدة. يهدف المشروع إلى سد الفجوة بين العمليات التقليدية للمقاهي والذكاء الاصطناعي الحديث. يتكون المشروع من موقع واجهة أمامية للعملاء، ومركز قيادة إداري شامل مدعوم بذكاء اصطناعي مزدوج المستويات.

### الأهداف الأساسية
- **الكفاءة التشغيلية:** تتبع المخزون في الوقت الفعلي مع خصم تلقائي مرتبط بالوصفات (Recipes).
- **الرقابة الاستراتيجية:** نظام موحد لسجلات التدقيق (Audit Logs) لضمان المساءلة الإدارية.
- **ذكاء الأعمال (BI):** ذكاء اصطناعي يقوم بتحليل بيانات قاعدة البيانات الحية وتقديم تقارير استراتيجية.
- **تجربة مستخدم متميزة:** واجهة عملاء تفاعلية، ثنائية اللغة (عربي/إنجليزي)، ومدعومة بالبحث الصوتي.

---

## 2. الحزمة التقنية (Tech Stack)

### **الواجهة الخلفية (Backend)**
- **البيئة التشغيلية:** Node.js v18+
- **الإطار البرمجي:** Express.js 5 (لإدارة المسارات والوسائط).
- **قاعدة البيانات:** MySQL 8.0 (لحفظ البيانات العلائقية).
- **محرك الذكاء الاصطناعي:** OpenAI GPT-4o-mini (متكامل عبر GitHub Models / Azure).
- **إدارة الملفات:** Multer (لرفع وحفظ صور المنتجات).
- **محرك التقارير:** jsPDF + AutoTable (لإنشاء تقارير PDF احترافية).

### **الواجهة الأمامية (Frontend)**
- **المكتبة الأساسية:** React.js 18
- **إدارة الحالة:** React Context API (لإدارة السلة، حالة المتجر، وتوثيق الأدمن).
- **المسارات:** React Router v7 (لهيكلية SPA).
- **التصميم:** Vanilla CSS + CSS Modules (جماليات Glassmorphism).
- **الحركات:** useReveal hook + CSS Keyframes.
- **التمرير الناعم:** Lenis.js

## 4. تحليل قاعدة البيانات (MySQL Schema)

يعتمد نظام CaffAIne على 14 جدولاً مترابطاً لضمان سلامة البيانات والتدقيق الاحترافي.

### **الجداول الرئيسية والعلاقات**
| الجدول | الغرض | الحقول الرئيسية |
| :--- | :--- | :--- |
| `orders` | سجل المعاملات الأساسي | الرقم، المبلغ الإجمالي، الحالة، نوع الطلب |
| `order_items` | المنتجات داخل كل طلب | رقم الطلب، رقم المنتج، الكمية، السعر |
| `menu_items` | كتالوج المنتجات | الاسم، السعر، رقم القسم، التوفر |
| `inventory` | مستويات المخزون | اسم المادة، الكمية، الحد الأدنى للتنبيه |
| `recipes` | ربط المكونات بالمنتج | رقم المنتج، رقم مادة المخزون، الكمية المطلوبة |
| `admin_logs` | **سجل التدقيق** | بريد الأدمن، الاسم، الإجراء المتخذ، التفاصيل |
| `addons` | إضافات المنتجات | الاسم، السعر، الارتباط بالمخزون |
| `careers` | الوظائف المتاحة | المسمى الوظيفي، النوع، الحالة |

### **هيكلية العمليات (Atomic Transactions)**
يستخدم النظام **المعاملات البرمجية (Transactions)** عند وضع الطلبات:
1. التحقق من توفر المخزون لكل المكونات في السلة.
2. إدخال سجل الطلب.
3. إدخال كافة منتجات الطلب.
4. خصم المخزون بناءً على روابط الوصفات (Recipes).
5. خصم المخزون للإضافات المختارة.
6. تأكيد العملية (Commit) في حال النجاح، أو التراجع (Rollback) في حال فشل أي خطوة لمنع تضارب البيانات.

## 5. دورة حياة البيانات (Data Flow)

### **تدفق طلب العميل**
1. **الاختيار:** يختار العميل المنتجات والإضافات عبر `Menu.js`.
2. **الدفع:** يقوم `CartContext.js` بحساب الإجماليات وإرسال بيانات العميل.
3. **API:** يتم إرسال طلب `POST /api/orders` إلى السيرفر.
4. **التحقق:** يقوم السيرفر بالتحقق من المخزون عبر جدول `recipes`.
5. **التنفيذ:** يظهر الطلب في لوحة التحكم لدى الأدمن مع مؤقت تنازلي حي.

### **تدفق سجلات التدقيق (Audit Flow)**
1. **الإجراء:** يقوم الأدمن بعملية حساسة (مثل حذف منتج).
2. **الوسيط (Middleware):** يقوم `server.js` باعتراض الطلب واستخراج هوية الأدمن.
3. **التسجيل:** يتم إدخال سجل في `admin_logs` يوضح من قام بالإجراء وماذا فعل وتوقيت ذلك.
4. **الرقابة:** يشاهد قائد الفريق هذا السجل في `LeaderDashboard.js`.

## 6. طبقة الذكاء الاصطناعي (Dual-Tier)

### **Sophie (باريستا الذكاء الاصطناعي للعملاء)**
- تقوم بقراءة المنيو الحي، العروض النشطة، وساعات العمل.
- تساعد العملاء في اختيار المنتجات والإجابة على الأسئلة الشائعة باللغتين.

### **CaffAIne (خبير ذكاء الأعمال للإدارة)**
- يقوم بتنفيذ 19 استعلاماً متوازياً لقاعدة البيانات (الإيرادات، التوجهات، مستويات المخزون، سجلات الأدمن).
- يمكن للقائد سؤاله عن أداء الفريق، أسباب انخفاض الإيرادات، أو أي تفاصيل استراتيجية.
